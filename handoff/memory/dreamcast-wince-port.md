Research project (started 2026-06): get a fuller Windows CE running on a real Sega Dreamcast (SH-4 / SH7091, 16MB RAM @ phys 0x0C000000 / cached 0x8C000000, PowerVR2, AICA, Maple, GD-ROM, optional BBA RTL8139C). Working dir `C:\dev\Dreamcast` (not a git repo).

Two assets:
- `C:\wcedreamcast` = official Sega "Dragon" Dreamcast WinCE SDK v2.1.0.49 = **Windows CE 2.12**, closed-source. Prebuilt kernel at `release/{retail,debug}/0winceos.bin` (packed NK ROM image). Stripped game runtime: multitasking kernel present, NO shell/explorer. Has inc/lib/samples + tools (DUMPNK, bingen, GD Workshop, VCCE).
- `Arquivotheca/WinCE-src_20201004` (GitHub, Oct-2020 MS leak). Branches WINCE300/420/500/600/700/800. Each is `PRIVATE/WINCEOS/...` OS source only — NO PUBLIC tree, NO compiler/build tools. **Key: `WINCE300/.../NK/KERNEL/SHX/` is the SuperH kernel with real `#if defined(SH4)` MMU/INTC/TMU code; SOURCES uses `EXEBASE=0x8C040000` (DC cached RAM base).**

Decisions:
- Target **CE 3.0** (leak floor), not 2.12. Better base, SH-4 kernel present.
- **Path A** (do first): run stock closed kernel + custom EXEs via makeimg + dc-load/BBA. **Path B** (the research): build CE 3.0 from leak, write DC BSP/OAL from scratch (no DC BSP exists), use KallistiOS as hardware reference, marry leak source to Platform Builder 3.0 (has Hitachi SHC compiler). **Path C** (NT 4.0 on SH-4): rejected — NT never had SH HAL/codegen; multi-year; 16MB below NT floor.
- OAL bring-up order: SCIF serial → TMU tick → MMU/cache → KITL → PVR framebuffer → Maple input → shell.

Update (2026-06-23): chose **Path B**. Workspace built at `C:\dev\Dreamcast`: `wince-src/` (WINCE300 shallow clone, single-branch), `dc-bsp/` (our git repo: oal/dreamcast, drivers, inc, files, cesysgen), `docs/01-findings.md` + `docs/02-toolchain-setup.md`, `README.md`.

Key facts learned:
- `DUMPNK` REJECTS `0winceos.bin` ("Invalid NK image header") — it's a **Sega-wrapped** image, not standard CE `.bin`: 0x800 header (magic `D6 1A`), load base `0x0C010000` (DC phys RAM+64KB), payload 0x1CC800; `ECEC` ROMHDR sig at off 2112. Stock 2.12 modules (extracted): full CE core (COREDLL, filesys.exe, gwes.exe, device.exe, ole32, winsock, schannel, sysstart.exe, **Qshell.exe**) + DC drivers (MAPLE, SH4SER, TIMER, DDRAW/DDHAL/DDI/D3DIM, DSOUND/SNDCORE, MPPP/microstk, WATAPI, wsegacd). GWES + minimal shell already present in 2.12.
- Leaked WINCE300 = **WINCEOS only** (NK incl SHX SH-4, CORE, DEVICE, FSDMGR, GWE/MGDI, COMM), ~143 src files, 5.4MB. **No PUBLIC tree, no shell, no build tools.** → NOT buildable alone.
- ~~Hard dependency: Platform Builder 3.0~~ **SUPERSEDED (2026-06-23).** PB 3.0 needs a CD key (won't pirate). Instead use **github.com/gweslab/WindowsCE-Build-Tools** — ships the MS **SH compiler** free (`bin/I386/SH/cl.exe` + c1/c1xx/c2.dll + `shasm.exe`), plus link/nmake/rc/lib and **OAK+DDK headers** (`ce3-oak/INC`). Compiles the leaked SHX kernel + OAL + drivers; no license. PB 3.0 now optional-only. Path B = leaked WINCEOS source ⊕ gweslab SH compiler ⊕ our dc-bsp.

It ships: loose kernel `nk.exe`/`nknodbg.exe`/`nkscifkd.exe`; ~90 loose OS module binaries in `release/{retail,debug}/OS|CoreOS_*` (coredll, gwes, filesys, ole32, **shell.exe**, IE stack, all DC drivers); full toolchain in `tools/` (**makeimg.exe, romimage.exe, fmerge.exe, bingen.exe**, regcomp, res2exe, rebaseic); full `.bib`/`.reg`/`.dat` (ce/common/config/platform/**gemini.bib** = 113 mods); `makeimg.out` build log. **KERNEL BUILD STARTED (2026-06-23, commit 23d2a84).** `build-kernel.bat` compiles leaked NK with SH cc. Key learnings: (1) MUST use pure CE3 include chain, NOT the 2.12 DC SDK inc (it redefines DWORD/BOOL, lacks CE3 consts). Chain: `bsp\inc` ; leak `NK\INC` ; `NK\KERNEL\SHX` ; `CORE\INC` ; gweslab `ce3-oak\INC` (OAK) ; gweslab **`ce3-ppc2k\include`** (base SDK windows/winnt/winbase/windef — had to sparse-add ce3-ppc2k+ce3-hpc2k, ~290M). Defines `-DSH4 -DSHx -DUNDER_CE=300 -D_WIN32_WCE=300 -DUNICODE -DKERNEL`. (2) **Leaked SHx headers are an INCOMPLETE snapshot**: `MEM_SHX.H` missing `VA_SECTION` (other CPUs' MEM_*.H have it); `SECTION_SHIFT`/`CURTLSPTR_OFFSET`/`KINFO_OFFSET` defined nowhere in leak. **Reconstructed in `bsp\inc\mem_shx_patch.h`** (force-included `/FImem_shx_patch.h`): VA_SECTION=25, SECTION_SHIFT=25, CURTLSPTR_OFFSET=0x000 (KSSHX.H lpvTls), KINFO_OFFSET=0x300 (KSSHX.H PendEvents 0x340 = aInfo[KINX_PENDEVENTS=16]). KERNEL.H ERRFALSE asserts re-validate vs C struct. Cleared all 4; errors 50→35.
